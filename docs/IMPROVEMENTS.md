# IMPROVEMENTS — 自研创新点

## 创新点 1：Agentic 轨迹细粒度错误分析器

- 传统/朴素基线：人工分析 10 条失败轨迹，平均耗时 30 分钟，准确率 70%。
- 做法：定义错误分类 Schema，自动解析每个步骤的 tool selection、tool args、knowledge miss、reasoning gap、state loss。
- 目标变化：分析时间↓93%，准确率↑28.6%。
- 可编码实现：`backend/agents/error_analyzer.py`。
- 测量方法：使用 50 条预标注失败轨迹对比人工标签，计算分类准确率与耗时。

## 创新点 2：基于失败轨迹的 DPO 数据自动生成

- 传统/朴素基线：人工构建 100 条 DPO 偏好数据对，成本约 2000 元，耗时 1 天。
- 做法：优化 Agent 将失败轨迹改写为 `chosen`，保留失败输出为 `rejected`，生成偏好对。
- 目标变化：成本↓99.75%，时间↓99.3%。
- 可编码实现：`backend/agents/dpo_generator.py`。
- 测量方法：对比人工标注和自动生成数据在下游 eval 成功率上的提升。

## STAR-R 绑定

- 因为实现了错误分析器，所以错误分析时间从 30 分钟/10 条目标降至 2 分钟/10 条（目标↓93.3%）。
- 因为实现了失败轨迹 DPO 生成器，所以偏好数据构建成本从 2000 元/100 对目标降至 5 元/100 对（目标↓99.75%）。
- 因为用自动数据进行 QLoRA DPO，所以特定 Agent 任务成功率目标↑20%，需要实测后回填。
